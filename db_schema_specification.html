Database Schema & Workflow Specification

1. Document Lifecycle & Approval Pipeline

1
Document Ingestion: Scan files via OCR, extract key-value data, and determine type using the LLM engine.
2
Data Verification Desk: AP Executive reviews extracted metadata and corrects variances.
3
Gate Entry Verification: Cross-reference invoice items with a physical Goods Receipt (GRN).
4
Decision: Is the PO in the Master Table?
YES Path: NO / EMPTY Path:
PO Invoice Workflow
  1. Indenter Approval
  2. PO Owner Approval
  3. Department Head Approval
Non-PO Standard Flow
  1. Cost Center Manager
  2. Head of Finance
5
Local Server Archival: Permanently save documents in local disk `/uploads/` and flag for ERP Sync.

2. Data Dictionary Specifications

2.1 User / Employee Table (User)

Field Name Data Type Key Description
id String (UUID) PK Unique identifier for the user.
username String UNIQUE Unique username used for local credential login.
employee_id String UNIQUE Corporate employee ID (e.g. EMP-1003).
email String UNIQUE Email address (used for notifications and logins).
role String - System role (e.g., admin, employee).
name String - Display name of the employee (e.g., Vijay Kumar).
department String (Nullable) - Department name.

2.2 Invoice Table (Invoice)

Field Name Data Type Key Description
id String (UUID) PK Unique invoice ID.
invoice_number String - Vendor invoice number.
vendor_name String - Extracted name of the supplier/vendor.
po_number String - PO number associated with the invoice.
amount Float - Total invoice amount.
status String - Main status (e.g. Data Verification Pending).
file_path String - Path to the document stored on the local server disk.

2.3 PO Details Table (CorporateAppMock)

Field Name Data Type Key Description
po_number String PK Unique PO identifier.
po_owner_email String (Nullable) - Email of the PO owner (approver token: [PO_OWNER]).
indenter_email String (Nullable) - Email of the indenter/requester (approver token: [INDENTER]).
dept_head_email String (Nullable) - Email of the department head (approver token: [DEPT_HEAD]).
